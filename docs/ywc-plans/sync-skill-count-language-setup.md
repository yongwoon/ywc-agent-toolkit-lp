## Purpose

`ywc-agent-toolkit` (the upstream toolkit this landing page markets) has an **open, unmerged** PR — [#125](https://github.com/yongwoon/ywc-agent-toolkit/pull/125) — that adds a **language-setup skill** (`ywc-setup-language` for Claude Code, `ywc-setup` for Codex) letting users persist a default output language, and, per its current diff, bumps the Codex-only skill count in the toolkit's own README from 41 to 42 while leaving Claude Code at 41.

**Critical precondition verified during planning**: as of this spec's authoring, `gh pr view 125 --repo yongwoon/ywc-agent-toolkit` reports `state: OPEN`, `mergedAt: null`. A live directory count against `claude-code/skills/` and `codex/skills/` on the toolkit's `main` branch (excluding the non-skill `scripts/` directory) shows **41 skills on both tools today** — i.e., the landing page's current "41 skills" claim is accurate *right now*, and the divergence described above does not yet exist upstream. This spec's numeric assumptions (Codex 42 / Claude Code 41) are therefore provisional, taken from an in-flight, unmerged diff, not from merged reality — **every numeric edit in this spec is gated on re-verifying this at execution time (FR-1)**.

This spec exists to make the landing page (LP) ready to absorb that upstream change the moment it merges, and separately to fix an already-existing, unrelated arithmetic mismatch in the Feature Grid category counts (discovered during investigation) that predates PR #125 entirely.

Add one new `featureGrid.items` category — label "Toolkit & Setup" (or the executor's closest natural-language equivalent per locale) — sized as follows:
