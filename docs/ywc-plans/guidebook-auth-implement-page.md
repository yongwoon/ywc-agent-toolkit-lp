## Purpose

`ywc-agent-toolkit` (the upstream toolkit this landing page's Guidebook documents) merged [PR #144](https://github.com/yongwoon/ywc-agent-toolkit/pull/144) (`mergedAt: 2026-07-16T05:54:48Z`, confirmed via `gh pr view 144 --repo yongwoon/ywc-agent-toolkit --json state,mergedAt`), adding a new `ywc-auth-implement` skill to both Claude Code and Codex. Unlike the sibling infra-page spec's upstream PR (#131, open at plan time), this skill is already on `main` — no soft dependency on a future merge exists for this spec.

`ywc-auth-implement` is a single skill, but its internal flow is multi-phase and high-stakes enough to warrant a dedicated Guidebook page rather than a one-paragraph catalog entry: a preflight gate (branch reuse, `.env.example` placeholders, stack-evidence routing, existing-auth hard stop, legal-draft labeling), a 9-category policy interview (sign-in method, MFA, session handling, password reset/hashing boundary, profile fields, account deletion, shallow RBAC, consent, abuse prevention), a dynamic (never-hardcoded) library/managed-service recommendation, dispatch to three existing named agents (`ywc-backend-coder`, `ywc-frontend-coder`, `ywc-doc-writer`) under TDD discipline, and a security-audit + policy-conditional-E2E gate before a non-blocking PR suggestion.

This depth is structurally comparable to the Guidebook's existing **single-skill dedicated pages** — page 10 (`10-e2e-test-strategy.md`, 611 words, one skill) and page 11 (`11-design-review.md`, 414 words, two skills) — rather than the 4-skill pipeline pages (13, 14). Per the user's stated preference during planning ("상황별 가이드에 사용방법을 자세히 기술하는게 좋다"), this spec creates a **new dedicated Workflow Guide page**, sized similarly to pages 10/11 (roughly 500-700 words, not page 14's 1024-word 4-skill depth), rather than expanding the existing "Full Skill Reference" catalog entry.
