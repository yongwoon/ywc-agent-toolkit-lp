ywc-plan

Claude Code Codex Planning & Spec

Convert a rough idea into an implementation-ready artifact — a directly executable plan.md (Small) or a spec document (Medium / Large). ywc-plan auto-assesses scale and routes to the right downstream skill.

#Overview

ywc-plan is the pre-planning entry point of the pipeline. Given a natural-language change request, it clarifies intent, investigates the codebase, judges the change's scale, and produces the appropriate planning artifact — then hands off to the next skill. It never executes work itself; you decide when to proceed.

#When to use

  • You want a plan for a feature but there's no spec and no tasks yet.
  • You're unsure whether a change is a Small fix or needs a full spec.
  • You need to prepare the spec that ywc-task-generator will consume.

#Usage

Invoke as a slash-command with your request, or just describe the change in natural language:

claude code · codex
/ywc-plan "add notification settings to the profile page"

#Input & output

Input is a natural-language change request. Output depends on the assessed scale:

ScaleOutput
Small./plan.md — a directly executable single-PR plan
Medium / Largedocs/ywc-plans/<slug>.md — a spec consumed by ywc-spec-validate and ywc-task-generator

Each path prints an explicit handoff message naming the next step.

#How it works

Five steps, in order:

  • Clarify — asks the four anchors at once: What / Why / Out of Scope / Done When.
  • Investigate — reads only the key files (CLAUDE.md, package.json, docs/architecture/).
  • Assess scale — picks exactly one of Small / Medium / Large (defaults to Medium if ambiguous).
  • Branch — writes plan.md for Small, or a spec document for Medium / Large.
  • Handoff — names the next skill; you decide whether to run it.

#Safety invariants

The scale automatically escalates to Medium or higher when the change involves:

! Database migrations or schema changes · introducing a new library or framework · a new externally-exposed API contract · authentication / authorization logic · cross-cutting changes spanning two or more modules.

#Examples

Small change → plan.md

/ywc-plan "fix the copy button tooltip on mobile"
# → Small · wrote ./plan.md · next: /ywc-code-gen

Larger change → spec

/ywc-plan "add OAuth login across web and mobile"
# → Large · wrote docs/ywc-plans/oauth-login.md
# next: /ywc-spec-ready  (or /ywc-spec-validate)
